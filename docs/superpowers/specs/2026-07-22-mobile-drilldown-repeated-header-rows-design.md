# 모바일 드릴다운 반복 헤더 행 설계

> 상태: 그릴링 반영 승인됨
>
> 작성일: 2026-07-22
>
> 선행 설계: `2026-07-21-mobile-drilldown-original-row-design.md`

## 1. 배경

`드릴다운 후 선택 행 원본`은 모바일 목차에서 응답 행을 선택한 뒤 해당 행만 원본 열 배치로 보여준다. 일부 설문 표는 정식 테이블 헤더 대신 본문의 한 행 또는 여러 행을 시각적 헤더로 작성한다. 현재 분류기는 이런 행을 일반 항목으로 취급하므로 `항목 0/0`처럼 응답할 수 없는 카드가 목차에 나타난다.

운영자는 숫자 범위로 정식 헤더와 본문 행을 선택해 모든 상세 화면 상단에 반복할 수 있어야 한다.

## 2. 목표

- 정식 테이블 헤더와 작성된 본문 행을 하나의 연속 범위 문법으로 선택한다.
- 선택된 본문 행을 목차와 진행률에서 제외한다.
- 선택된 헤더 블록을 모든 선택 행 상세에 같은 열 배치로 반복한다.
- 기존 `상세에서 제외할 앞쪽 열 수`와 함께 사용할 수 있다.
- 일반 테이블과 설명 테이블 소스의 radio·checkbox 드릴다운에 같은 규칙을 적용한다.

## 3. 비목표

- 비연속 행 선택
- 체크박스 기반 행 선택 UI
- 반복 범위에 입력 셀이 있는지 검사
- 반복 범위가 모든 응답 행을 차지하는지 검사
- 행 삭제·이동 뒤 범위가 가리키는 의미를 보존하거나 경고
- 동적 생성 행을 반복 헤더로 선택
- 섹션별 반복 범위
- 반복 헤더의 세로 sticky 고정
- 반복 헤더 입력 셀을 필수·조합 검증에서 제외
- 데스크톱 표 렌더링 변경

반복 범위에 입력 행을 포함하거나 모든 본문 행을 포함한 결과는 운영자가 테스트 과정에서 확인한다.

## 4. 범위 문법

행 번호 `0`은 정식 테이블 헤더 전체를 나타낸다. 본문 행은 작성된 `tableRowsData` 순서를 기준으로 1부터 센다.

| 입력 | 의미 |
| --- | --- |
| 빈 값 | 반복 헤더 없음 |
| `0` | 정식 테이블 헤더 전체 반복 |
| `3` | 본문 3행만 반복 |
| `2-3` | 본문 2행부터 3행까지 반복 |
| `0-2` | 정식 테이블 헤더 전체와 본문 1~2행 반복 |

빈 값 또는 공백을 제거한 `숫자`, `숫자-숫자` 형식만 정상 입력이다. 시작 번호가 끝 번호보다 큰 값과 음수는 정상 범위로 저장하지 않는다. 입력 중에는 문자열을 그대로 유지하고 `Enter` 또는 포커스를 벗어날 때 확정한다. 잘못된 형식이면 직전에 저장된 정상 범위로 되돌린다.

명시적인 `null/null`은 반복 헤더 없음이다. 필드가 모두 없는 과거 스냅샷은 기본 범위 `0-0`으로 해석한다. 저장된 값이 정수가 아니거나 음수이거나 시작 번호가 끝 번호보다 크거나 두 필드 중 하나만 `null`이면 비정상 값이다. 비정상 값은 `0-0`으로 폴백한다.

현재 본문 행 수를 벗어난 번호는 자동 보정하지 않는다. 실제로 존재하는 작성 행만 범위에 매칭하며 설정 문자열은 그대로 유지한다. 행을 재정렬하면 범위는 행 ID가 아니라 재정렬된 현재 작성 위치를 따라간다.

## 5. 설정 UI

설정은 `모바일 표시 방식`이 `드릴다운 후 선택 행 원본`일 때만 노출한다.

- 라벨: `상세에서 반복할 헤더 행`
- 도움말: `비우면 반복하지 않습니다. 0은 진짜 헤더이며, 3 또는 0-2처럼 입력합니다.`
- 입력 컨트롤: 단일 text input
- 기본 표시값: `0`

기존 `상세에서 제외할 앞쪽 열 수` 입력은 그대로 유지하며 두 설정은 서로 독립적이다.
다른 모바일 표시 방식으로 전환하면 입력을 숨기되 저장값은 보존하고, 다시 드릴다운 모드로 돌아오면 빈 값 또는 이전 범위를 복원한다.

## 6. 저장 모델

화면에는 범위 문자열 하나를 보여주지만 정본 데이터에는 시작·끝 정수를 저장한다.

```ts
interface Question {
  mobileDrilldownRepeatHeaderStartRow?: number | null;
  mobileDrilldownRepeatHeaderEndRow?: number | null;
}
```

DB 컬럼도 integer 두 개를 사용한다.

- `questions.mobile_drilldown_repeat_header_start_row integer default 0`
- `questions.mobile_drilldown_repeat_header_end_row integer default 0`

두 컬럼은 nullable이다. 신규 질문과 기존 질문은 `0/0`으로 시작하고, 입력을 지우면 `null/null`을 명시적으로 저장한다. 빈 상태는 질문 생성·수정·복제·전체 저장·보관함 적용·버전 스냅샷에서도 `null/null`로 보존한다.

두 필드는 기존 모바일 표시 설정의 영속 경로를 따른다. 실제 마이그레이션 번호는 구현 시작 시 다른 worktree의 최신 마이그레이션을 확인한 뒤 배정한다.

## 7. 목차와 진행률

1. 정상 범위를 해석한다. 빈 값이면 행을 제외하지 않는다.
2. 범위에 포함된 1 이상의 번호를 작성된 본문 행 ID로 변환한다.
3. 현재 표시되는 해당 행을 모바일 분류기에 전달할 목차용 행 목록에서 제외한다.
4. 전체·섹션 진행률도 제외된 행을 세지 않는다.

따라서 반복 헤더 행은 `0/0` 카드나 빈 섹션으로 나타나지 않는다. 원본 `displayRows`는 상세 투영에 필요하므로 삭제하지 않고 목차용 목록만 별도로 만든다. 반복 행 제거로 rowspan 앵커가 빠지면 기존 visible-row 재계산 규칙으로 첫 남은 행에 앵커 정체성과 표시 속성을 승격한 뒤 분류한다.

범위는 질문 전체에 하나만 적용한다. 조건부 표시로 반복 행이 숨겨져도 작성 행 번호를 다시 매기지 않는다.

## 8. 상세 렌더링

상세 화면은 다음 순서로 구성한다.

1. 범위에 `0`이 있으면 정식 헤더 전체를 투영한다.
2. 범위에 포함된 현재 표시 가능한 본문 행을 작성 순서대로 투영한다.
3. 선택한 응답 행을 투영한다.

정식 헤더는 `tableHeaderGrid`가 있으면 다단 헤더 전체다. `tableHeaderGrid`가 없으면 `tableColumns[].label`로 만든 기존 단일 헤더 폴백을 사용한다. 헤더 그리드가 없고 열 라벨도 모두 비어 있으면 `0`만 건너뛴다.

반복 헤더와 선택 행에는 동일한 규칙을 적용한다.

- 현재 표시 가능한 열만 유지
- `상세에서 제외할 앞쪽 열 수` 적용
- colspan·rowspan 재계산
- 셀 `mobileDisplay="hidden"` 처리
- 같은 가로 스크롤 위치 사용

본문 반복 행은 원본 높이·병합·배경·텍스트 배치를 유지하는 표시 전용 행이다. 입력 컨트롤이 있으면 원래 모양을 유지하되 비활성화한다. 그 아래의 선택 행만 응답 입력 행으로 동작한다. 반복 입력 셀을 필수·조합 검증에서 제외하는 예외는 만들지 않는다.

반복 헤더는 상세 표의 바로 위에 함께 렌더링되며 세로 sticky로 고정하지 않는다. 상세 항목 사이를 이동할 때 기존 가로 스크롤 위치를 유지하고 반복 헤더와 선택 행이 같은 위치로 움직인다.

반복 헤더와 선택 행 경계를 가로지르는 세로 병합은 경계에서 분리한다. 반복 헤더 안에서 끝나는 rowspan은 유지하고, 선택 행에는 기존 continuation materialize 규칙으로 필요한 앵커 내용을 복원한다.

### 8.1 열 제목 숨기기와의 우선순위

범위에 `0`이 포함되면 모바일 선택 행 상세에서는 정식 헤더를 명시적으로 표시한다. 이 경우에만 `hideColumnLabels`보다 반복 헤더 설정이 우선한다. 데스크톱과 다른 모바일 표시 모드의 `hideColumnLabels` 동작은 바꾸지 않는다.

## 9. 최소 폴백 규칙

추가 방어 로직은 다음 네 가지로 제한한다.

1. 조건부 표시로 반복 본문 행이 숨겨지면 해당 행만 상세 헤더에서 생략한다.
2. 헤더 그리드와 열 라벨 폴백이 모두 없는 표에서 범위에 `0`이 있으면 `0`만 건너뛰고 나머지 본문 반복 행을 표시한다.
3. 저장된 시작·끝 값이 비정상이면 `0`으로 폴백하며 본문 행은 목차에서 제거하지 않는다.
4. 범위에 `0`이 포함되면 모바일 선택 행 상세에서만 정식 헤더 표시가 `hideColumnLabels`보다 우선한다.

본문 행 수를 벗어난 정상 범위는 자동 보정하지 않는다. 예를 들어 본문이 5행일 때 `8-10`은 반복 행을 만들지 않고 본문 행도 목차에서 제거하지 않으며 설정값을 유지한다. 모든 본문 행을 반복 범위에 포함하면 빈 목차와 `0/0` 진행률을 그대로 표시한다.

## 10. 데이터 흐름과 책임

### 설정

```text
MobileTableDisplaySettings
  → 범위 문자열 파싱
  → DynamicTableEditor
  → Question의 start/end 갱신
  → 기존 질문 저장·스냅샷 경로
```

범위 문자열의 파싱·포맷·폴백은 별도 순수 유틸리티가 소유한다. 컴포넌트가 범위 규칙을 중복 구현하지 않는다.

### 응답 렌더링

```text
Question start/end
  → 반복할 작성 행 ID 계산
  ├─ 빈 값이면 기존 행 전체를 목차에 전달
  ├─ 목차용 displayRows에서 반복 행 제외
  └─ 상세 투영에 반복 헤더 블록 전달
       → 정식 헤더
       → 반복 본문 행
       → 선택 응답 행
```

기존 `projectMobileOriginalRow` 계층이 열 제외·병합·숨김 투영을 계속 소유한다. 표준 테이블과 choice 테이블은 같은 범위 해석 유틸리티와 투영 결과를 사용한다.

## 11. 테스트

### 순수 로직

- 빈 값, `0`, `3`, `2-3`, `0-2` 파싱과 포맷
- 잘못 저장된 범위의 `0` 폴백
- 범위 밖 번호를 자동 보정하지 않음
- 작성 행 번호를 행 ID로 변환

### UI와 영속성

- 해당 모바일 모드에서만 입력 노출
- Enter·blur 범위 확정과 잘못된 입력 시 직전 값 유지
- 빈 값 `null/null`과 질문 저장·복제·보관함·버전 스냅샷 왕복
- 모바일 표시 모드를 바꿨다가 돌아와도 범위 보존

### 모바일 렌더링

- 반복 본문 행이 목차와 진행률에서 제외됨
- 빈 값, `0`, `3`, `2-3`, `0-2` 상세 순서
- 다단 헤더 전체와 열 라벨 단일 헤더 폴백
- 앞쪽 열 제외와 반복 헤더의 조합
- colspan·rowspan·숨김 셀 투영
- 반복 헤더와 선택 행 경계의 rowspan 분리
- 반복 입력 컨트롤의 원본 모양·비활성 상태
- 조건부로 숨겨진 반복 행 생략
- 헤더 그리드·열 라벨이 모두 없는 `0-2`
- `0` 지정 시 모바일 상세의 `hideColumnLabels` 재정의
- 일반 테이블과 choice 테이블의 동일 동작
- 헤더와 선택 행의 가로 스크롤 동기화·항목 간 위치 유지·목차 복귀 초기화

## 12. 완료 기준

- 운영자가 단일 범위 입력으로 반복 없음·정식 헤더·본문 행을 선택할 수 있다.
- 반복 본문 행은 목차와 진행률에서 사라진다.
- 모든 선택 행 상세에서 지정된 헤더 블록과 응답 행의 열이 정렬된다.
- 기존 열 제외·병합·숨김·조건부 표시 규칙이 유지된다.
- 명시한 네 가지 폴백 외의 추가 검증이나 자동 보정은 도입하지 않는다.
